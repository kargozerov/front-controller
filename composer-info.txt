1. Через терминал зайти в папку с проектом
2. Выполнить команду 'composer init' (либо 'php composer.phar init') - инициализирует проект, как composer-пакет
    Необходимо будет ответить на следующие вопросы:
    1) Package name (имя пакета) - имя разработчика и название проекта.
        Если устраивает предложенное название, нажимаем Enter,
        если нет - указываем свои латиницей в формате разработчик/название_проекта
    2) Description (описание проекта). Можно пропустить и нажать Enter, либо написать несколько слов о проекте
    3) Author (данные автора). Можно пропустить, нажать n и нажать Enter,
        либо указать в формате разработчик <example@mail.ru>
    4) Minimum Stability (фильтр загружаемых библиотек по версиям).
        Обычно указывают один из вариантов: stable, RC, beta, alpha, dev.
    5) Package Type (тип пакета). Нужно выбрать один из предлагаемых вариантов и написать его
    6) License (лицензия, по которой будет распространяться пакет). Нужно указать один из вариантов:
        Apache-2.0, BSD, GPL, LGPL, MIT
        Посмотрите варианты https://getcomposer.org/doc/04-schema.md#license.
    7) Вопрос по зависимостям (пишите no и Enter). Зависимости пока не устанавливаем
    8) Еще вопрос по зависимостям (пишите no и Enter). Зависимости пока не устанавливаем
    9) Do you confirm generation. Если сформированный json устраивает, нажимаем Enter
3. Команда 'composer update' (либо 'php composer.phar update') - выполнять каждый раз, когда вносились изменения в composer.json файл
4. Необходимо настроить автозагрузку классов.
    В файле composer.json нужно добавить
    "autoload": {
            "psr-4": {
                "Vendor\\ProjectName\\": "src/"
            }
        }

     , где Vendor - разработчик,
           ProjectName - имя проекта
           src/ - имя папки, на которую будет распространяться автозагрузка классов (в проекте с занятия - папка src/)
5.  Команда 'composer dump-autoload' (либо 'php composer.phar dump-autoload') - выполнять каждый раз,
    когда вносились изменения в настройки автозагрузки классов в composer.json файле
6. В index.php файле нужно подключить autoload.php, чтобы автозагрузка классов работала.
require_once __DIR__ . '/../vendor/autoload.php';

Для каждого класса в папке src определить пространство имен (по сути путь от папки src до файла с классом).
Например, для класса Router, который лежит в src/Core:
namespace Vendor\ProjectName\Core\;
Теперь для подключения данного класса не нужно использовать require,
достаточно написать use Vendor\ProjectName\Core\Router;
